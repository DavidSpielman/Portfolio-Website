Webcam MIDI Controller

  • date: July 2021 - Present

In my free time I enjoy composing music. The inspiration for this project came about as I was brainstrorming interersting ways to make and play music. When I was younger, I remember enjoying outdoor interactive music installationsat children's science museums. I wanted to recreate the satisfaction of striking a chime or PVC pipe with a paddle at home, so I began working on this MIDI controller.

This project incorporates OpenCV, ROS, a Teensy 4.0 microcontroller, a Logitech Pro 9000 USB webcam, the Reaper DAW (Digital Audio Workstation), and a table tennis paddle. OpenCV tracks the position of the table tennis paddle on the live video footage displayed by the webcam, and depending on where the paddle is placed on the screen, a ROS publisher node will publish note data to a subscriber node associated to the Teensy microcontroller. The Teensy microcontroller will then send MIDI data to the Reaper DAW where it will be converted into recordable sound using audio plugins.

To track the position of the table tennis paddle I needed to use a computer vision library. I decided to go with OpenCV since it is open source and is widely used. This was my first exposure to computer vision and OpenCV, so I was excited to start learning through this project.

After about a week and a half of reading through the OpenCV documentation, troubleshooting installation and video capture issues, and consulting many forums and YouTube videos, I learned enough to create the tracker below. It uses OpenCV's built-in CSRT tracker which I found to work best for this application. I placed a Post-It note on the table tennis paddle to improve tracking. I also added a reset function to readjust the selected region of interest (ROI) if the tracker lost the object being tracked. In the GIF below, I selected the ROI to be the table tennis paddle, and then I reset the ROI to track my face.

Once the tracker was ready, I added an GUI interface on the video displayed by the webcam to help the user select which note interval they wished to play. Based on the location of the center of the ROI (the red dot in the gif below) OpenCV will publish which interval the user has selected through a ROS publisher node.

The ROS publisher node sends messages through rosserial which is a way for ROS to communicate with Arduino compatible microcontrollers. I chose to use a Teensy 4.0 microcontroller since it can natively send MIDI data through USB, is Arduino IDE compatible, and due to its small size. I have made a ROS subscriber node that works with the Teensy microcontroller, but due to issues with a previous Teensy microcontroller that was defective, I am still working on ways to send MIDI data from the subscriber node to my DAW using a new Teensy microcontroller. (DAW stands for Digital Audio Workstation and it is a software tool used by many musicians to manipulate, record, and process sound to make and release music.)

The next step for this project is to send MIDI data from the Teensy microcontroller to the Reaper DAW. I also plan on adding an IMU to the table tennis paddle to translate how hard the GUI boxes are "hit" to MIDI velocity data. This will allow the user to control the volume of the notes produced by how fast they move the paddle.
